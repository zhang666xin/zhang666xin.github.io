2.引用类型转换：

（1）向上类型转换（隐式/自动类型转换),是小类型到大类型的转换。

（2）向下类型转换（强制类型转换），是大类型到小类型的转换。

eg： Dog dog=new Dog()

Animal animal=dog； 而 Dog dog2=animal; 有风险；Dog dog2=(Dog)animal; 进行强制类型转换。但是Cat cat=(Cat)animal;错误，类型不一样，此类型为Dog类型。解决：instanceof运算符避免类型转换的安全性问题。

用法：

　　boolean result = object instanceof class

参数：

　　result ：boolean类型。

　　object ：必选项。任意对象表达式。

　　class：必选项。任意已定义的对象类。

说明：

　　如果该object 是该class的一个实例，那么返回true。如果该object 不是该class的一个实例，或者object是null，则返回false。

3.抽象类：

（1）语法定义：使用abstract关键字修饰。eg：public abstract void call（）{…};

（2）作用：限制规定子类必须实现某些方法，但不关注实现细节。

（3）使用规则：抽象类中可以包含普通的方法，也可以没有抽象方法，且抽象类不能直接创建，可以定义引用变量。

4.接口：

（1）概念：某一批类所需要遵守的规范，它只规定这些类里必须提供某些方法。

（2）语法：[修饰符] interface 接口名 [extends 父接口1,父接口2…]

​ {

​ 0-n个常量定义…

​ 0-n个抽象方法的定义…

​ }

​ 接口是用来被继承、被实现的，修饰符一般建议用public，不能使用private和protected修饰。

常量：接口属性是常量，即使定义时不加public static final，系统也会自己加上。

抽象方法：定义时不加public abstract修饰符，系统也会自己加上。

（3）使用：一个类可以实现一个或多个接口，实现接口用implements关键字，（右键包->new->interface->起名，一般在名字前加一个I[若还想继承其他父接口，点击add])java一个类只能继承一个父类，通过实现多个接口可以做补充。

三：异常与异常处理

1.异常5个关键字： try catch finally throw throws。。。

try 要把有可能出现异常的代码放到try代码块中。

catch 是捕获和处理异常。

finall 不管有没有异常都要执行的代码，可加可不加finally。

throw 在方法中手动抛出异常。

throws 声明方法有可能抛出的各种异常。

2.java所有异常类都继承在Throwable类中，该类下包括error（“程序终结者”比如虚拟机坏了）和exception（编码、环境、用户输入出现问题）

在exception类下又包括RuntimeException(非检查异常)和检查异常（文件异常、SQL异常）。

在RuntimeException下又包括NullPointerException(空指针异常)、ArrayIndexOutOfBoundsException(数组下标越界异常)、ClassCastException(类型转换异常)、ArithmeticException(算术异常)等。。。

3.try-catch 代码块三种情况：

try{

}catch{

}

a.无异常。则执行完try后不执行catch

b.有异常，被捕获。执行完try后在执行catch

c.有异常，未捕获。程序中断运行。

使用多重catch语句时，注意顺序。异常子类一定要位于异常父类（exception）之前，因为系统会按照就近原则，先子类后父类。
